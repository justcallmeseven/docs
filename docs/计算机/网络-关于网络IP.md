## IP地址后面斜杠加具体数字详解

> **简介：** 192.168.1.100/16、192.168.1.100/24，是一种叫做 **CIDR（无类别域间路由选择）** 的IP地址表示方法。简单来说，斜杠后的数字**代表了子网掩码中二进制“1”的个数，也就是IP地址中“网络号”所占的位数**。它决定了这个IP地址属于哪个网络，以及这个网络里能容纳多少台设备。

### 🧐 `/16` 与 `/24` 的区别

为了更清晰地对比，我们以 `192.168.1.100` 这个IP为例，看看它分别配上 `/16` 和 `/24` 后有什么不同：

| 特性           | **192.168.1.100/16**              | **192.168.1.100/24**            |
| :------------- | :-------------------------------- | :------------------------------ |
| **子网掩码**   | `255.255.0.0`                     | `255.255.255.0`                 |
| **网络位数**   | 前 **16** 位是网络号              | 前 **24** 位是网络号            |
| **主机位数**   | 后 **16** 位是主机号              | 后 **8** 位是主机号             |
| **所属网络**   | `192.168.0.0`                     | `192.168.1.0`                   |
| **可用IP范围** | `192.168.0.1` ~ `192.168.255.254` | `192.168.1.1` ~ `192.168.1.254` |
| **广播地址**   | `192.168.255.255`                 | `192.168.1.255`                 |
| **可用主机数** | **65,534** 个                     | **254** 个                      |

简单来说，`/16` 代表一个**大型网络**，`/24` 则是从这个大网络中划分出的一个**小型子网**

### 🤔 斜杠数字的奥秘：CIDR表示法

这种表示法的核心是 **CIDR（Classless Inter-Domain Routing）**，即无类别域间路由选择。它彻底改变了传统IP地址的分类方式。

**1. 历史由来：解决IP地址短缺的“良方”**
在CIDR出现之前，IP地址被划分为固定的A、B、C等类别。这种“有类”的划分方式非常不灵活，导致大量IP地址被浪费，加剧了IPv4地址的枯竭。

为了解决这个问题，**CIDR于1993年被正式引入**。它抛弃了固定的地址类别，允许将IP地址空间更灵活地划分为任意大小的“块”或“子网”。

**2. 本质：子网掩码的“速记法”**
CIDR的本质就是**子网掩码的简化写法**。斜杠后的数字（如`24`）直接告诉你子网掩码中有多少个连续的“1”。

- **子网掩码** 是一个32位的二进制数，用于区分IP地址中的“网络号”和“主机号”。
- 子网掩码中，**“1”对应网络号**，**“0”对应主机号**。
- 例如，`/24` 就表示子网掩码的前24位是1，后8位是0，即二进制的 `11111111.11111111.11111111.00000000`，换算成十进制就是 `255.255.255.0`

### 📝 其他常见的CIDR表示法

除了 `/16` 和 `/24`，还有很多类似的写法，它们都遵循同样的规则：

- **`/8`**：对应子网掩码 `255.0.0.0`。常用于A类网络或超大型网络。
- **`/25`**：对应 `255.255.255.128`。将一个 `/24` 网络再一分为二，每个子网有126个可用IP。
- **`/26`**：对应 `255.255.255.192`。每个子网有62个可用IP。
- **`/27`**：对应 `255.255.255.224`。每个子网有30个可用IP。
- **`/28`**：对应 `255.255.255.240`。每个子网有14个可用IP。
- **`/29`**：对应 `255.255.255.248`。每个子网有6个可用IP。
- **`/30`**：对应 `255.255.255.252`。常用于路由器之间的点对点连接，每个子网只有2个可用IP。
- **`/32`**：对应 `255.255.255.255`。代表一个**单独的、特定的IP地址**。
- **`/0`**：对应 `0.0.0.0`。代表**所有IP地址**，即整个互联网。

CIDR表示法非常强大，它用一个简洁的格式就表达了IP地址及其所属的网络范围，是现代网络配置和管理的基础。















---



> 其实这种形式就是用CIDR（**无类别域间路由选择**，Classless and Subnet Address Extensions and Supernetting））的形式表示的一个网段，或者说子网。

我们知道确定一个子网需要知道主机地址和子网掩码，但用CIDR的形式，可以简单得到两个数值。举例说吧，192.168.0.0/24”就表示，这个网段的IP地址从192.168.0.1开始，到192.168.0.254结束（192.168.0.0和192.168.0.255有特殊含义，不能用作IP地址）；子网掩码是255.255.255.0。

上面的子网掩码怎么来的呢？其实关键就在“24”上。我们知道IP地址是四个十进制数组成的，相当于32位二进制。用CIDR表示形式，后一个数字将这32位进行了间隔（以24为例）：前24位用"1"表示，后面8位用0表示，得到一个二进制数：

11111111 11111111 11111111 00000000。将其转化为十进制，就是：255.255.255.0了。

例如：192.168.1.0/24表示网段是192.168.1.0，子网掩码是24位，子网掩码为：255.255.255.0，用二进制表示为：11111111 11111111 11111111 00000000 ，这里为什么是24呢，就是因为子网掩码里面的前面连续的“1”的个数为24个，一定要连续的才行。

再给你举个例子，192.168.1.0/28表示的意思是网段是192.168.1.0，子网掩码为：255.255.255.240，用二进制表示为：11111111 11111111 11111111 11110000。

这时候你也许就疑惑了，就是24和28两个字不一样，为什么网段是一样的呢？

24位说明网络位是24位，那么主机位就是32-24=8位了，则子网的IP个数是254个，即是从00000001到11111110.

28位说明网络位是28位，那么主机位4位，则子网的IP个数是14个，即是从00000001到00001110.
