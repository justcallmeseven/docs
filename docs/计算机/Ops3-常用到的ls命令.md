`ls` 是 Linux 运维中最常用的命令之一，用于列出目录内容。虽然看起来简单，但结合不同参数，它能发挥出非常强大的作用。下面按功能分类整理运维工作中最常用的 `ls` 命令。

------

## 一、基础用法

| 命令                 | 作用                                   | 说明                                           |
| :------------------- | :------------------------------------- | :--------------------------------------------- |
| `ls`                 | 列出当前目录下的文件和文件夹           | 不显示隐藏文件（以 `.` 开头的文件）            |
| `ls /path/to/dir`    | 列出指定目录的内容                     | 例如 `ls /etc`                                 |
| `ls -a`              | 列出**所有**文件，包括隐藏文件         | `-a` = all，会显示 `.` 和 `..` 等              |
| `ls -A`              | 列出几乎所有文件，但不显示 `.` 和 `..` | 比 `-a` 更清爽，隐藏文件仍然显示               |
| `ls -l`              | 以**长格式**显示详细信息               | **最常用组合之一**，显示权限、大小、修改时间等 |
| `ls -lh`             | 以**人类可读**的格式显示文件大小       | `-h` = human-readable，自动转为 KB、MB、GB     |
| `ls -la` 或 `ls -al` | 以长格式显示**所有**文件（含隐藏）     | **运维最常用组合**                             |

> ```
> username@seaserversname:~$ ls -la
> total 44
> drwxr-x--- 5 username username 4096 Jun  5 09:58 .
> drwxr-xr-x 3 root     root     4096 Jun  4 03:30 ..
> -rw------- 1 username username 6627 Jun 16 08:17 .bash_history
> -rw-r--r-- 1 username username  220 Mar 31  2024 .bash_logout
> -rw-r--r-- 1 username username 3771 Mar 31  2024 .bashrc
> drwx------ 2 username username 4096 Jun  4 03:42 .cache
> drwx------ 3 username username 4096 Jun  4 05:50 .config
> -rw------- 1 username username   20 Jun  5 09:58 .lesshst
> -rw-r--r-- 1 username username  807 Mar 31  2024 .profile
> drwx------ 2 username username 4096 Jun  4 03:36 .ssh
> -rw-r--r-- 1 username username    0 Jun  4 09:13 .sudo_as_admin_successful
> ```
>
> #### **`ls` 的颜色高亮功能**，颜色代表文件类型，不是随机显示的
>
> #### 你看到的蓝色
>
> 例如：
>
> ```
> .cache
> .config
> .ssh
> ```
>
> 显示为蓝色。
>
> 因为它们是：
>
> ```
> d
> ```
>
> 即：
>
> ```
> Directory（目录、文件夹）
> ```
>
> 从你的输出：
>
> ```
> drwx------ 2 username username 4096 Jun 4 03:42 .cache
> ^
> ```
>
> 第一位是：
>
> ```
> d
> ```
>
> 表示目录，所以显示为蓝色。
>
> #### 你看到的白色
>
> 例如：
>
> ```
> .bashrc
> .profile
> .bash_history
> ```
>
> 显示为白色。
>
> 因为它们是：
>
> ```
> -
> ```
>
> 即：
>
> ```
> Regular File（普通文件）
> ```
>
> 从输出：
>
> ```
> -rw-r--r-- 1 username username 3771 Mar 31 2024 .bashrc
> ^
> ```
>
> 第一位：
>
> ```
> -
> ```
>
> 表示普通文件，所以一般显示白色。
>
> #### 常见颜色对照
>
> | 颜色     | 类型                           | 第一列         |
> | -------- | ------------------------------ | -------------- |
> | 蓝色     | 目录                           | d              |
> | 白色     | 普通文件                       | -              |
> | 绿色     | 可执行文件                     | -（带 x 权限） |
> | 青色     | 符号链接                       | l              |
> | 黄色黑底 | 设备文件                       | b 或 c         |
> | 红色     | 压缩包或特殊文件（某些发行版） | 视配置而定     |
>
> #### 为什么 Linux 要用颜色？
>
> 因为运维经常需要一眼区分：
>
> ```
> 白色 → 普通文件
> 蓝色 → 目录
> 绿色 → 可执行程序
> 青色 → 链接
> ```
>
> 例如：
>
> ```
> cd .ssh
> ```
>
> 你一看到：
>
> ```
> .ssh
> ```
>
> 是蓝色，就知道：
>
> ```
> 这是目录，可以 cd 进去
> ```
>
> 如果：
>
> ```
> .bashrc
> ```
>
> 是白色，就知道：
>
> ```
> 这是文件，不能 cd
> ```
>
> ------
>
> #### 可以验证一下
>
> 执行：
>
> ```
> ls -l --color=always
> ```
>
> 或者：
>
> ```
> dircolors -p | less
> ```
>
> 能看到系统定义的颜色规则。
>
> ------
>
> #### 一个运维小技巧
>
> 以后执行：
>
> ```
> ls
> ```
>
> 先看颜色，再看名字：
>
> ```
> 蓝色   → 可以 cd
> 白色   → 可以 cat/nano
> 绿色   → 可以直接执行
> 青色   → 指向其他文件
> ```
>
> 很多时候，不用敲：
>
> ```
> ls -l
> ```
>
> 仅靠颜色就能大致判断文件类型。

### 示例：`ls -lh` 输出

```
$ ls -lh

drwxr-xr-x 2 root root 4.0K Jun 16 09:00 config/
-rwxr-xr-x 1 root root  12K Jun 15 22:10 script.sh
```

## 二、排序类（运维排错非常有用）

| 命令      | 作用                                   | 使用场景                     |
| :-------- | :------------------------------------- | :--------------------------- |
| `ls -lt`  | 按**修改时间**排序，**最新**的在前     | 查看最近修改的日志或配置文件 |
| `ls -ltr` | 按修改时间排序，**最旧**的在前（反向） | 查看最早的日志或历史文件     |
| `ls -lS`  | 按**文件大小**排序，**最大**的在前     | 查找占用空间最大的文件       |
| `ls -lSr` | 按文件大小排序，**最小**的在前（反向） | 查找小文件                   |
| `ls -lX`  | 按**扩展名**（文件类型）排序           | 按文件类型归类               |

> #### `ls -lt` 默认按什么排序？
>
> 按的是：
>
> ```
> mtime
> Modification Time
> 文件内容最后修改时间
> ```
>
> 不是：
>
> ```
> 改文件名时间
> 创建时间
> 访问时间
> ```
>
> #### Linux 实际有三种时间
>
> 可以查看：
>
> ```
> stat 50-cloud-init.yaml.bak
> ```
>
> 会看到类似：
>
> ```
> Access: 2026-06-05 08:35:00
> Modify: 2026-06-05 08:35:00
> Change: 2026-06-18 17:30:00
> ```
>
> ------
>
> #### Access（atime）
>
> ```
> 最后访问时间
> ```
>
> 例如：
>
> ```
> cat file
> ```
>
> ------
>
> #### Modify（mtime）
>
> ```
> 最后修改文件内容时间
> ```
>
> 例如：
>
> ```
> nano file
> ```
>
> 保存后改变。
>
> ------
>
> #### Change（ctime）
>
> ```
> 最后状态变更时间
> ```
>
> 包括：
>
> - 改文件名
> - chmod
> - chown
> - 改链接数
> - 修改内容
>
> 都会更新 ctime。

### 实用场景举例

```
# 查看当前目录下最新的5个日志文件
ls -lt | head -5

# 查找最大的3个文件
ls -lS | head -3

# 查看最近修改过的配置文件
ls -lt /etc/*.conf | head -10
```

## 三、筛选与过滤类

| 命令                | 作用                           | 说明                               |
| :------------------ | :----------------------------- | :--------------------------------- |
| `ls *.log`          | 只列出 `.log` 结尾的文件       | 通配符匹配                         |
| `ls -d */`          | 只列出**目录**，不显示目录内容 | `-d` 不进入目录，`*/` 匹配所有目录 |
| `ls -R`             | **递归**列出子目录的内容       | 查看整个目录树，小心输出过多       |
| `ls -l | grep "^d"` | 只显示目录（通过过滤权限位）   | 组合使用 `grep` 进行精细筛选       |

## 四、显示详细信息类

| 命令                 | 作用                                                 | 说明                                |
| :------------------- | :--------------------------------------------------- | :---------------------------------- |
| `ls -l`              | 长格式，显示权限、链接数、所有者、组、大小、修改时间 | 核心命令                            |
| `ls -lh`             | 长格式 + 人类可读大小                                | 最常用                              |
| `ls -li`             | 显示文件的 **inode 号**                              | `-i` 显示文件在文件系统中的唯一编号 |
| `ls -n`              | 以**UID/GID**数字形式显示所有者和组                  | 代替用户名显示数字ID                |
| `ls -l --time=atime` | 显示**访问时间**（atime）而不是修改时间              | 较少用                              |
| `ls -l --time=ctime` | 显示**状态变更时间**（ctime）                        | 权限或元数据变更时间                |

### 长格式输出解读

```
-rw-r--r-- 1 root root 2.3M Jun 16 10:30 access.log
```



| 字段           | 含义                                           |
| :------------- | :--------------------------------------------- |
| `-rw-r--r--`   | 文件类型和权限（`-` 文件，`d` 目录，`l` 链接） |
| `1`            | 硬链接数                                       |
| `root`         | 文件所有者                                     |
| `root`         | 文件所属组                                     |
| `2.3M`         | 文件大小                                       |
| `Jun 16 10:30` | 最后修改时间                                   |
| `access.log`   | 文件名                                         |

## 六、实用技巧

### 1️⃣ 统计目录下的文件数量

```
ls -1 | wc -l        # 统计当前目录下的文件数（不含隐藏文件）
ls -1a | wc -l       # 统计所有文件（含隐藏）
```



### 2️⃣ 查看目录占用大小（与 `du` 配合）

```
ls -lh               # 仅显示文件大小，不统计目录大小
du -sh *             # 统计每个子目录/文件的实际磁盘占用
```



### 3️⃣ 彩色输出

默认 `ls` 会用不同颜色区分文件类型（目录、可执行文件、链接等）。如果颜色不显示，可以用：

```
ls --color=auto      # 或直接 `ls` 在多数系统默认带颜色
```



### 4️⃣ 查看特定类型的文件

```
ls -lh *.log         # 所有日志文件
ls -lh /etc/*.conf   # 所有 .conf 配置文件
```



### 5️⃣ 结合 `find` 使用（更强大）

`ls` 适合快速查看，`find` 适合复杂条件搜索：

```
find /var/log -name "*.log" -type f -mtime -7 -ls  # 7天内修改的日志
```

## 💡 别名增强效率

在 `~/.bashrc` 或 `~/.bash_aliases` 中添加别名，可以大幅提高效率：

```
alias ll='ls -alh'      # 所有文件，人类可读大小
alias l='ls -lh'        # 长格式，人类可读
alias lt='ls -lt'       # 按时间排序
alias lS='ls -lS'       # 按大小排序
alias l.='ls -d .*'     # 只显示隐藏文件
```

添加后执行 `source ~/.bashrc` 生效。这样每天只需敲两三个字母就能完成大量操作。
