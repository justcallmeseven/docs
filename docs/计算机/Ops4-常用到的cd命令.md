在Linux运维中，`cd`（Change Directory）命令本身**没有任何复杂的参数**（不像`ls`有几十种选项），它最核心的功力体现在**对路径快捷键的极致运用**上。

`cd`命令的用法可以分为四个层次：**基础路径**、**灵魂快捷键**、**目录栈（高级技巧）**和**符号链接处理**。

------

### 一、基础路径用法

这是最根本的用法，决定了你去哪里。

| 命令            | 作用                               | 说明                                     |
| :-------------- | :--------------------------------- | :--------------------------------------- |
| `cd /`          | 切换到**根目录**                   | Linux文件系统的顶层                      |
| `cd /etc/nginx` | 切换到**绝对路径**                 | 无论当前在哪，直接跳转到指定完整路径     |
| `cd logs`       | 切换到**相对路径**                 | 进入当前目录下的 `logs` 子目录           |
| `cd ..`         | 切换到**上级目录**                 | 返回上一层（`..` 代表父目录）            |
| `cd ../..`      | 切换到**上两级目录**               | 连续返回两层                             |
| `cd ~` 或 `cd`  | 切换到**当前用户的家目录**         | `~` 代表家目录，输入`cd`直接回车效果相同 |
| `cd ~/projects` | 切换到**家目录下的projects文件夹** | 从任何位置直接回到自己的工作目录         |

------

### 二、⭐ 灵魂快捷键（运维高手必备）

这些快捷键能让你在频繁切换目录时节省大量时间。

| 命令       | 作用                             | 实战场景                                                     |
| :--------- | :------------------------------- | :----------------------------------------------------------- |
| **`cd -`** | 切换到**上一个**所在目录（返回） | 你在 `/var/log/nginx` 看日志，切到 `/etc/nginx` 改配置，改完想立刻回日志目录，输入 `cd -` 即可瞬间返回。 |
| `cd !$`    | 将上一个命令的参数作为目录并进入 | 你刚 `cat /etc/hosts`，接着想进入 `/etc`，输入 `cd !$`（`!$` 代表上一个命令的最后一个参数 `/etc/hosts` 的路径部分，其实是文件，会报错，更常用的是配合 `mkdir`：`mkdir test` 然后 `cd !$` 直接进入刚建的目录）。 |

> **强烈推荐**：把 `cd -` 刻在肌肉记忆里，排错时在“日志目录”和“配置目录”之间来回穿梭极其丝滑。

------

### 三、高级运维技巧：目录栈（`pushd` / `popd`）

如果 `cd -` 只能让你在两个目录间来回，那么**目录栈**可以让你在多个常用目录间“存书签”并自由跳转。

| 命令             | 作用                               | 说明                                           |
| :--------------- | :--------------------------------- | :--------------------------------------------- |
| `pushd /var/log` | 切换到该目录，并将当前目录压入栈顶 | 相当于“收藏当前路径，并跳转到新路径”           |
| `pushd`          | 不加参数，交换栈顶前两个目录的位置 | 在最近两个目录间快速切换（类似`cd -`的增强版） |
| `popd`           | 弹出栈顶目录，并切换到弹出的目录   | 相当于“从书签里取出最近收藏的路径并返回”       |
| `dirs`           | 显示当前的目录栈内容               | 查看自己收藏了哪些路径                         |

**实战场景**：你需要在 `/etc/nginx`（配置）、`/var/log/nginx`（日志）、`/opt/app`（代码）三个目录间反复操作。

bash

```
# 收藏当前目录并跳转到日志目录
pushd /var/log/nginx   
# 收藏日志目录并跳转到配置目录
pushd /etc/nginx      

dirs  # 显示：/etc/nginx /var/log/nginx ~

popd  # 弹出了 /etc/nginx，回到了 /var/log/nginx
popd  # 弹出了 /var/log/nginx，回到了最初的 ~
```

### 四、处理符号链接（软链接）的高级参数

`cd` 命令有两个处理软链接的参数，在排查路径时非常关键。

| 命令               | 作用                                     | 说明                           |
| :----------------- | :--------------------------------------- | :----------------------------- |
| `cd -P /path/link` | **物理**路径，显示链接指向的真实路径     | 进入软链接指向的实际目录       |
| `cd -L /path/link` | **逻辑**路径，显示链接自身的路径（默认） | 就待在软链接名称下（默认行为） |

**实战场景**：服务器上 `/var/www` 是一个指向 `/data/webapp/public` 的软链接。

bash

```
cd /var/www
pwd   # 默认显示 /var/www （逻辑路径）

cd -P /var/www
pwd   # 显示 /data/webapp/public （物理真实路径）
```



在运维排错时，如果你怀疑链接指向错误，用 `cd -P` 直达真实目录查看实际内容非常有用。

### 五、实用组合技巧

| 技巧                       | 命令示例                    | 作用                                                         |
| :------------------------- | :-------------------------- | :----------------------------------------------------------- |
| **直接进入上一个参数目录** | `cd $_`                     | `$_` 代表上一个命令的最后一个参数。如果你刚 `ls /etc/nginx/sites-enabled/`，执行 `cd $_` 直接进入该目录。 |
| **结合管道返回上级**       | `cd -`                      | 强烈推荐，不需要额外组合。                                   |
| **带错误抑制的跳转**       | `cd /some/path 2>/dev/null` | 脚本中常用，防止报错信息刷屏。                               |

### 六、总结速查表

| 你的需求                 | 输入的命令            |
| :----------------------- | :-------------------- |
| 去根目录                 | `cd /`                |
| 去当前用户的家           | `cd` 或 `cd ~`        |
| 返回上一级               | `cd ..`               |
| **返回上一次所在的目录** | **`cd -`** (重中之重) |
| 去某个绝对路径           | `cd /etc/nginx`       |
| 进入刚创建的新目录       | `mkdir test && cd $_` |
| 查看目录栈（书签）       | `dirs`                |
| 压入栈并跳转             | `pushd /tmp`          |
| 弹出栈返回               | `popd`                |
| 进入软链接的真实路径     | `cd -P /path/to/link` |

### 💡 终极大招：`CDPATH` 环境变量

如果某个目录（比如 `/var/log`）你每天要进去几十次，可以在 `~/.bashrc` 中配置：

bash

```
export CDPATH="/var/log:/etc:/opt"
```
