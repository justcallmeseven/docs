## 常用到的ip命令

`ip` 命令是 Linux 下网络配置的核心工具，功能非常强大。下面是运维工作中最常用的一些 `ip` 命令，按功能分类整理。

> **小提示**：几乎所有 `ip` 命令都可以用简写。比如 `addr` 可以简写为 `a`，`link` 为 `l`，`route` 为 `r`。为了便于记忆，下表会先列出完整命令，括号中标注简写。

------

### 一、查看信息类（最常用）

| 用途                 | 命令 (完整)         | 命令 (常用简写)      | 作用说明                                                     |
| :------------------- | :------------------ | :------------------- | :----------------------------------------------------------- |
| **查看IP地址**       | `ip addr show`      | `ip a`               | 查看所有网卡的IP地址、子网掩码、广播地址、状态。**这是最常用的命令。** |
| **查看单个网卡**     | `ip addr show eth0` | `ip a show eth0`     | 只查看指定网卡（如 eth0）的IP信息。                          |
| **查看路由表**       | `ip route show`     | `ip r` 或 `ip route` | 查看系统的路由表，包括默认网关。                             |
| **查看ARP缓存**      | `ip neigh show`     | `ip n`               | 查看邻居（ARP）表，即局域网内IP与MAC地址的对应关系。         |
| **查看网络接口状态** | `ip link show`      | `ip l`               | 查看所有网卡的链路层状态（如 UP/DOWN）、MAC地址、数据包统计等。 |

------

### 二、IP地址配置（临时修改，重启网络或系统后失效）

| 用途           | 命令示例                                     | 作用说明                                                     |
| :------------- | :------------------------------------------- | :----------------------------------------------------------- |
| **添加IP地址** | `sudo ip addr add 192.168.1.100/24 dev eth0` | 给 eth0 添加一个额外的IP地址。注意要加上子网掩码（如 `/24`）。 |
| **删除IP地址** | `sudo ip addr del 192.168.1.100/24 dev eth0` | 删除之前添加的IP地址。                                       |
| **清空所有IP** | `sudo ip addr flush dev eth0`                | 一次性清空 eth0 上配置的所有IP地址。                         |

#### 永久增加一个额外 IP：

#### 第一步：备份配置

```
sudo cp /etc/netplan/50-cloud-init.yaml /etc/netplan/50-cloud-init.yaml.bak
```

#### 第二步：编辑配置

```
sudo nano /etc/netplan/50-cloud-init.yaml
```

改成：

```
network:
  version: 2
  ethernets:
    ens33:
      dhcp4: true
      addresses:
        - 192.168.44.168/24
```

注意缩进必须保持一致（YAML 对缩进非常敏感）。建议：列表统一缩进 **2个空格**

> 动态主机配置协议 DHCP（Dynamic Host Configuration Protocol，动态主机配置协议）
>
> DHCP 服务器可以理解成网络里的 **“自动分配 IP 地址的管理员”**。
>
> 如果没有 DHCP，你每台电脑、手机、服务器都要手工设置：
>
> ```
> IP地址
> 子网掩码
> 网关
> DNS
> ```
>
> 非常麻烦。
>
> 有了 DHCP 后，设备接入网络时会自动申请这些信息。
>
> ------
>
> #### 举个现实例子
>
> 你把电脑接入网络：
>
> ```
> 电脑
>  ↓
> "有人能给我一个IP吗？"
> ```
>
> DHCP 服务器收到请求后：
>
> ```
> 好的
> 
> 给你：
> IP：192.168.44.134
> 掩码：255.255.255.0
> 网关：192.168.44.2
> DNS：xxxx
> ```
>
> 于是电脑就能上网了。
>
> 整个过程自动完成。
>
> #### 当前的情况
>
> 你的配置：
>
> ```
> dhcp4: true
> ```
>
> 表示：
>
> ```
> 开机时自动找DHCP服务器要IP
> ```
>
> 所以：
>
> ```
> 192.168.44.134
> ```
>
> 不是你配置的，
>
> 而是 DHCP 服务器分配的。
>
> 从输出可以看出：
>
> ```
> inet 192.168.44.134/24 dynamic
> ```
>
> 其中：
>
> ```
> dynamic = DHCP分配
> ```
>
> ------
>
> #### DHCP 服务器通常是谁？
>
> #### 家庭网络
>
> ```
> 光猫
>    ↓
> 路由器
>    ↓
> 电脑
> 手机
> 电视
> ```
>
> 一般是路由器充当 DHCP 服务器。
>
> 例如：
>
> - ASUS 路由器
> - TP-Link 路由器
> - Xiaomi 路由器
>
> 都默认开启 DHCP 服务。
>
> ------
>
> #### VMware NAT 网络
>
> 你的情况很像这个：
>
> ```
> Windows主机
>    ↓
> VMware NAT
>    ↓
> Ubuntu虚拟机
> ```
>
> 在这种模式下：
>
> ```
> VMware DHCP Server
> ```
>
> 负责发放 IP。
>
> 例如：
>
> ```
> 192.168.44.134
> 192.168.44.135
> 192.168.44.136
> ```
>
> 都是它分配的
>
> #### DHCP 的工作流程
>
> 简单来说是四步：
>
> ```
> 1. Discover
> "谁是DHCP服务器？"
> 
> 2. Offer
> "我是，我给你192.168.44.134"
> 
> 3. Request
> "我要这个IP"
> 
> 4. Ack
> "批准"
> ```
>
> 简称：
>
> ```
> DORA
> Discover
> Offer
> Request
> Ack
> ```
>
> 这是面试里经常问的知识点。
>
> #### 你的服务器为什么总是拿到 .134
>
> 因为 DHCP 会记住你的网卡 MAC：
>
> ```
> 00:0c:29:5c:je:88
> ```
>
> 它会记录：
>
> ```
> MAC
> ↓
> 00:0c:29:5c:je:88
> 
> IP
> ↓
> 192.168.44.134
> ```
>
> 所以下次开机：
>
> ```
> 哦，是老客户
> 继续给你 .134
> ```
>
> 这叫：
>
> ```
> DHCP Lease（租约）
> ```
>
> #### 怎么查看 DHCP 服务器是谁？
>
> 在你的 Ubuntu 上执行：
>
> ```
> nmcli dev show ens33 | grep DHCP
> ```
>
> 或者：
>
> ```
> journalctl -u systemd-networkd | grep DHCP
> ```
>
> 如果是在 VMware NAT 网络里，经常会看到类似：
>
> ```
> DHCP Server: 192.168.44.254
> ```
>
> 或者其他地址。
>
> 你也可以直接执行：
>
> ```
> networkctl status ens33
> ```

#### 第三步：检查配置语法

先不要直接应用。

执行：

```
sudo netplan generate
```

如果没有任何输出，说明语法正确。

#### 第四步：应用配置

```
sudo netplan apply
```

然后查看：

```
ip addr show ens33
```

正常应该看到：

```
inet 192.168.44.134/24 scope global dynamic ens33
inet 192.168.44.168/24 scope global secondary ens33
```

这样：

```
192.168.44.134  ← DHCP获取
192.168.44.168  ← 永久附加
```

重启后仍然存在。

---

#### 更安全的方式（推荐远程服务器）

如果你是通过 SSH 操作服务器，建议先执行：

```
sudo netplan try
```

而不是 `apply`。

`netplan try` 会给你大约 120 秒确认时间：

```
Do you want to keep these settings?
```

- 输入 `ENTER` 确认 → 配置永久生效
- 如果网络断开，没有确认 → 自动回滚

这样即使配置错误，也不会把自己锁在服务器外面。

### 三、网络接口控制（启动/关闭）

| 用途         | 命令示例                         | 作用说明                                    |
| :----------- | :------------------------------- | :------------------------------------------ |
| **启动网卡** | `sudo ip link set eth0 up`       | 相当于 `ifconfig eth0 up`，激活网络接口。   |
| **关闭网卡** | `sudo ip link set eth0 down`     | 相当于 `ifconfig eth0 down`，停用网络接口。 |
| **修改MTU**  | `sudo ip link set eth0 mtu 9000` | 修改网卡的MTU值（最大传输单元）。           |

> `p` 命令里有 **很多子命令**：
>
> ```
> link, addr, route, neigh, rule, tunnel ...
> ```
>
> - 对于 **查询类命令**，Linux 支持**唯一前缀缩写**。
> - `l` 唯一匹配 `link`，所以可以直接写 `ip l`。
> - 其他查询子命令也一样，比如 `ip a` → `ip addr`，`ip r` → `ip route`。
>
> #### `ip link set` 修改时不行的原因
>
> 修改命令一般是：
>
> ```
> ip link set dev ens33 mtu 1400
> ```
>
> - `set` 是 **动作命令**（subcommand），它有自己的语法规则。
> - Linux 的解析器在 **修改/动作模式下** 不允许随意缩写，**必须完整写出关键字**。
>
> #### 原因本质
>
> Linux 命令解析器区分两类：
>
> 1. **只读查询类**：用户友好 → 支持缩写
> 2. **修改/管理类**：语法严格 → 不支持缩写，避免误操作
>
> 换句话说：**查询可以随便读，修改要小心**
>
> 
>
> **奇怪：“sudo ip l set ens33 mtu 1000”多试几下也可以？？？**

### 四、路由配置（临时修改，重启失效）

| 用途             | 命令示例                                        | 作用说明                                                 |
| :--------------- | :---------------------------------------------- | :------------------------------------------------------- |
| **添加默认网关** | `sudo ip route add default via 192.168.1.1`     | 设置默认网关为 192.168.1.1。                             |
| **添加静态路由** | `sudo ip route add 10.0.0.0/24 via 192.168.1.2` | 发往 10.0.0.0/24 网段的数据，走 192.168.1.2 这个下一跳。 |
| **删除默认网关** | `sudo ip route del default`                     | 删除默认路由。                                           |
| **删除静态路由** | `sudo ip route del 10.0.0.0/24`                 | 删除指定的静态路由。                                     |

### 五、调试与排错类

| 用途             | 命令示例               | 作用说明                                                     |
| :--------------- | :--------------------- | :----------------------------------------------------------- |
| **监控网络事件** | `ip monitor`           | 实时显示系统网络配置的变化（如IP变更、路由变化等），非常适合排错。 |

### 核心总结表（速查）

| 你想做什么          | 就用这个命令（简写版）          |
| :------------------ | :------------------------------ |
| **查看IP**          | `ip a`                          |
| **查看路由/网关**   | `ip r`                          |
| **查看ARP/MAC地址** | `ip n`                          |
| **查看网卡状态**    | `ip l`                          |
| **启动/关闭网卡**   | `ip l set eth0 up/down`         |
| **添加/删除IP**     | `ip a add/del IP/掩码 dev eth0` |

| **查看统计信息**    | `ip -s l show eth0`             |

------

### 💡 实战小窍门

1. **永远记得加 `sudo`**：修改配置（add/del/set）都需要 root 权限。
2. **这些修改是临时的**：重启网络服务或重启系统后，所有改动都会丢失。要让配置永久生效，需要在对应发行版的网络配置文件中修改（如 Ubuntu 的 Netplan，CentOS 的 ifcfg-* 文件）。
3. **善用简写**：运维效率很重要，`ip a` 和 `ip r` 是最常用的两个，一定要记牢。
